Chapter 1. Introduction

The UIMA Simple Server makes results of UIMA processing available in a simple, XML-based format. The intended use of the Simple Server is to provide UIMA analysis as a REST service. The Simple Server is implemented as a Java Servlet, and is deployed through the web application developed in the Syntatica UIMA Webpage example setup instructions.  This document covers the configuration of the UIMA processing pipes.

1.2. Configuring the Servlet

Finally, you need to update your web.xml descriptor. Edit the WebContent/WEB-INF/web.xml file in your sample project. Minimally, it should look like this:

<?xml version="1.0" encoding="UTF-8"?>

<web-app xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"

    xmlns="http://java.sun.com/xml/ns/javaee"

    xmlns:web="http://java.sun.com/xml/ns/javaee/web-app_2_5.xsd"

    xsi:schemaLocation="http://java.sun.com/xml/ns/javaee

      http://java.sun.com/xml/ns/javaee/web-app_2_5.xsd"

    id="WebApp_ID" version="2.5">

    <display-name>parts-of-speech</display-name>

    <!-- Declare simple server test servlet -->

    <servlet>

        <servlet-name>parts-of-speech</servlet-name>

        <servlet-class>

            org.apache.uima.simpleserver.servlet.SimpleServerServlet

        </servlet-class>

        <!-- Define the path to the tokenizer pear file -->

        <init-param>

            <param-name>DescriptorPath</param-name>

            <param-value>

                WEB-INF/resources/descriptors/test.xml

            </param-value>

        </init-param>

    </servlet>

    <!-- Declare the URL of the servlet -->

    <servlet-mapping>

        <servlet-name>parts-of-speech</servlet-name>

        <url-pattern>/</url-pattern>

    </servlet-mapping>

</web-app>

 

If you're doing all of this in Eclipse, and you have Tomcat configured as a server, you can now right-click your project and select "Run on server...". This will start Tomcat and deploy web app.

Alternatively, you can build a WAR file out of your project. In Eclipse, right-click your project, and select "Export/Web/WAR file". Save the WAR file to some location and deploy it in Tomcat (for example, using the admin web app that comes with Tomcat).

This is it, your first simple service should be ready to run. Make sure Tomcat is running and direct your browser to the following URL: http://localhost:8080/parts-of-speech. You should see something like this:

1.3.1. Primitive Analysis Engine Descriptor Files

1.3.1.1    WhitespaceTokenizer

Copy the WhitespaceTokenizer.xml file from WhitespaceTokenizer/desc/ to WebContent/WEB-INF/resources/descriptors/

Right-click on the file within the project and select Open with->Component Descriptor Editor

Select the Capabilities tab.  You will see that it does not have any inputs, but has outputs of SentenceAnnotation and TokenAnnotation.

1.3.1.2    HmmTagger

Copy the HmmTagger.xml file from Tagger/desc/ to WebContent/WEB-INF/resources/descriptors/

Right-click on the file within the project and select Open with->Component Descriptor Editor

Select the Capabilities tab.  You will see that it has inputs of SentenceAnnotation and TokenAnnotation.  The outputs are the same with the additional attribute of posTag.

1.3.2. Aggregate Analysis Engine Descriptor File

The aggregate descriptor is a container that is used to identify the flow of the processing of the analysis engines.  We will create the test.xml that is described in the web.xml configuration.

 

Within the WEB-INF/resources/descriptors folder, create the test.xml file by selecting File->New->Other

You will see the following wizard:

Under the UIMA folder, select Analysis Engine Descriptor File, click on Next.

Type in test.xml in the file name field and click-on Finish.  Test.xml will open in the component editor as seen below.

 

 

Click on the Aggregate radio button.  The following message will pop up.  Click on OK.

 

 

Switch to the Aggregate view.

Click on Add

Select Import By Location.

Find the parts-of-speech project in the list and expand through src->main->webapp->WEB-INF->resources->descriptors and select WhitespaceTokenizer.

Repeat the above process for adding HmmTagger.xml.

In the command window, run mvn clean install within the parts-of-speech project.

Take the parts-of-speech.war file from parts-of-speech/target/ folder and place it in the webapps directory for Tomcat.  When Tomcat is started up, then the war file will be extracted into the parts-of-speech folder within the webapps directory.
